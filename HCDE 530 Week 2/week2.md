# Week 2 Observations — Competency 2: Code Literacy and Documentation

## What I Learned About Code Literacy

This week I worked directly inside Cursor, going through Python code and using AI to understand specific functions and concepts. I learned how certain built-in functions like `.split()` and `len()` work, how loops iterate over data, and how formatted output is structured. I also set up Cursor as my coding environment and connected the project to GitHub — learning how to initialize a repo, commit changes, and push to remote.

## What I Learned About Documentation

Documentation serves different audiences. Comments inside code help explain the purpose of a variable, function, or list — they make the logic readable to anyone (including future you). Docstrings go further by describing what a function does and what it returns. README and markdown files provide richer, higher-level context — and importantly, they also help AI tools understand the project better and function more accurately within it.

## Challenges

Initial setup had some friction — pushing and committing to GitHub for the first time took some troubleshooting. Once the workflow clicked (git add → commit -m → push), it became straightforward. The learning curve was mostly in the tooling setup rather than the code itself.

## Reflection: Code Literacy in HCDE

AI makes coding fast and accessible, but that's exactly why understanding how code works matters more, not less. Experimenting with code yourself — even at a basic level — makes you a better designer or product manager. You can have more informed conversations with engineers, ask better questions, and make more grounded decisions. Code literacy isn't about writing everything from scratch; it's about understanding enough to direct and evaluate what's being built.
